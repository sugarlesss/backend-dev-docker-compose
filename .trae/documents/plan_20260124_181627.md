## MySQL 版本划分重构计划

### 目标
将 MySQL 按版本划分为 57 和 80 两个独立子目录，便于管理和切换不同版本。

### 实施步骤

#### 1. 创建新的目录结构
```
MySQL/
├── 57/
│   ├── conf/
│   │   ├── my.cnf (默认配置，1G Buffer Pool)
│   │   ├── my.cnf.2G
│   │   ├── my.cnf.4G
│   │   ├── my.cnf.8G
│   │   ├── my.cnf.16G
│   │   └── my.cnf.32G
│   ├── data/
│   ├── log/
│   └── init/
└── 80/
    ├── conf/
    │   ├── my.cnf (默认配置，1G Buffer Pool)
    │   ├── my.cnf.2G
    │   ├── my.cnf.4G
    │   ├── my.cnf.8G
    │   ├── my.cnf.16G
    │   └── my.cnf.32G
    ├── data/
    ├── log/
    └── init/
```

#### 2. 移动和重命名配置文件
- 将 `my.cnf.57.*` 移动到 `MySQL/57/conf/` 并重命名为 `my.cnf.*`
- 将 `my.cnf.80.*` 移动到 `MySQL/80/conf/` 并重命名为 `my.cnf.*`
- 在 `MySQL/57/conf/` 和 `MySQL/80/conf/` 创建默认的 `my.cnf`（链接到 1G 版本）

#### 3. 更新 docker-compose.yml
- 修改 MySQL 服务的 volume 映射，使用 `./MySQL/80/...` 路径
- 修改 MySQL 服务的 image 为 `mysql:8.0.44`
- 更新 container_name 为 `mysql-80`
- 保持端口 3306 不变
- 在注释中说明如何切换到 57 版本

#### 4. 更新 rebuild.sh 脚本
- 修改 `init_mysql()` 函数，初始化 `MySQL/80/` 目录结构
- 创建 `MySQL/57/` 目录结构（备用）

#### 5. 更新 README.md
- 更新 MySQL 配置说明，反映新的目录结构
- 添加版本切换说明（修改 docker-compose.yml 中的路径和镜像版本）
- 更新容器名称说明

### 主要变更点
- MySQL 配置按版本隔离到独立目录（57 和 80）
- 默认使用 80 版本
- 通过修改 docker-compose.yml 切换版本
- .env 文件保持不变
